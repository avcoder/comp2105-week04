CSS Transitions

A transition describes how a property should display change when given a different value.

(When to move) + (What to change) + (How to change)

transition-duration

in seconds or milliseconds: 3s or 3000ms

Though milliseconds are actually a lot more robust to use in production code

If an animation is too fast to be interprested, it's cognitive load benefits can be lost

  • 100 ms, feels instantaneous
  • 100 ms ~ 1s, feels connected / 250 ~ 300ms is the animation sweet spot
  • 1s ~ 10s, disconnected (Perhaps needs a play button?)

transition-timing-function

Optional property that "cushions" the transition

Default is set to ease.

Easing

Easing, also known as "cushioning" in studio animation, describes an animation's rate of change over time.

For example, deceleration is actually ease-out. Acceleration is ease-in

Forms the basis of sprite animation.

Why Sprites?

It enables you to break up an animation into steps rather than the usual tweened motion that we’re used to.


Which principle of animation does this remind you of?


Straight Ahead &
Pose to Pose

Starts at the first drawing and works drawing to drawing to the end of the scene / Charted with key drawings at intervals throughout the scene.

Straight ahead animation starts at the first drawing and works drawing to drawing to the end of a scene. You can lose size, volume, and proportions with this method, but it does have spontaneity and freshness. Fast, wild action scenes are done this way.

Pose to Pose is more planned out and charted with key drawings done at intervals throughout the scene. Size, volumes, and proportions are controlled better this way, as is the action.

Sprite animations are best for small, animated embellishments in web interfaces... Typically, these animations will be created first in animation software like After Effects or even drawn frame-by-frame in something like Illustrator

Example sprites

steps(x)

steps(x) is a timing function that splits a block of keyframes into x equal steps, then hops between them.



it cycles through each snapshot or frame, pausing on each based on how many steps you divided your total duration into. So instead of continuous motion, your animation is divided into a series of holds

Easing comparison - linear vs steps 


#green { 
  transition: transform 3s linear;
}

#blue {
  transition: transform 3s steps(3);
}

#green:active, #blue:active {
  transform: translateX(300px);
}

start vs end 


#green { 
  transition: 3s steps(3, start);
}

#blue {
  transition: 3s steps(3, end);
}

#green:active, #blue:active {
  transform: translateX(300px);
}

Mismatched frames vs steps

Image file that twitter uses for Like button 


#green { 
  background: url('like.png') 0px 0px;
  transition: 1s steps(29);
}

#green:active {
  background-position: -2800px 0px;
}

Notice how each frame is shown with no transition or easing in between. It's just a hard cut and hold to the next frame. The continuity of the animation is a result of how closely each frame is drawn to the one before it, and the tendency of our eyes and brain to want to make things appear to be moving smoothly. It's a pretty neat effect
